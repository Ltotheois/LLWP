General

LLWP is a software for using Loomis-Wood plots to efficiently assign lines in spectra. It is published under GNU GPL v3 as it is based on a GUI library with the same copyright.

The program does not require specific units, thus all units can be used. However, be aware that the output files will use the input units and therefore might not be sensible in the SPFIT/SPCAT universe.

If you want to acknowledge LLWP, please cite this webpage as LLWP Loomis-Wood Plots, Luis BONAH, https://ltotheois.github.io/LLWP

Installation

The program requires python version 3.6 or higher.

The required libraries are included in the .exe file but have to be installed manually for the .pyw file. Two scripts for windows and ubuntu are available to help you install the needed dependencies. The ubuntu version does not need any user support and will compile PyQt5, which is needed for the GUI, on its own.

Please check that your python version and pip are updated, as the script will not update them if they are already installed. This is done, so that other software, that might rely on specific versions, will not run into problems. Also be aware, that you might have to run the script as root.
The latest numpy versions are not supported due to some internal problems. If you should see error messages related to this, try reverting to numpy version 1.19.3.

The windows version only installs the correct python libraries, this means Python and pip have to be installed and PyQt5 has to be compiled manually. The easiest way is to go to their website and download the newest version, the executable will compile PyQt5 for you.

The download folder also provides sample data for testing the program. The spectrum is a downsampled version of ethyl cyanide's spectrum, which was recorded in the Schlemmer Group @ University of Cologne. The .cat file is from the Cologne Database for Molecular Spectroscopy.

User Interface

The centerpiece of the program is the plot canvas with its subplots. Above the plot is a toolbar with controls for navigating the plot, and even further up is the menubar. The subplots center frequencies can be provided by different methods, which can be selected in the Define Series window. To change the appearance of the plots use the Configure Plots window, it provides options for the scaling, annotations and more. When a line is assigned, it is added to the Assigned Lines window, which also allows to edit the values and to delete all or specific assignments.

Additionally, there is a Log window for notifications, and windows for additional features, which can be accessed by selecting Modules in the menu bar.

The user interface can be set up in three different modes, being horizontal, vertical and window. To do so, please refer to the Flags section. horizontal places the plot canvas on the left side over the full height and puts the other windows to the right. In vertical mode the plot canvas takes the full width with the other windows being beneath. window mode detaches the other windows from the main window, so they can be freely positioned.

Also note, that the width and height of the windows can be changed by dragging the dividers with the mouse.

Loading Data and accepted Filetypes

The first step to using LLWP is loading data. The software distinguishes between three different kinds of files, being spectrum files, .cat files and .lin files.

Spectrum files are experimental measurements. By default they are tab separated and the frequencies are in the first column, the intensities in the second. To change the default behaviour refer to the Flags section. The program expects data with upwards facing peaks.

.cat files are catalogue files, they hold predictions for transitions. The format is according to the SPCAT specifications.

.lin files hold already assigned lines, so that they can be marked in the plots and the format is according to the SPFIT specifications.

For the three file types, there are two actions each, which are located in the menubar under File. The load action replaces the old files, whereas add keeps already loaded files of the same type.

To delete specific files, hide them or to customize the colours, refer to the three actions in the menu Files > Edit Spectrum Files, Files > Edit Cat Files and Files > Edit Lin Files. Additionally, there is an option to turn the spectra upside down, as all transitions should be indicated by positive peaks.

Additionally, project files can be saved and loaded. They are identified by the .llwp extension and have the same structure as .ini files. A project consists of the paths to all spectrum, .cat and .lin files as well as the current dataset in the Assigned Lines window.

Define a Series to follow

The Define Series tab provides three different methods to define a series. Additionally, it allows to set a specific number of plots and a width which will be applied by clicking Apply. These values are ignored, if Ignore on apply is checked. This is useful to certain users, as it is a quick way to changing these values. The lowest subplot is the first in the series, usually being the lowest in energy.

Expression allows to enter an expression depending on J and J0 (written as J and J_0). Then the value for J0 can be changed with the second input field. Please note, that the expression is evaluated with pythons eval command. An exemplary command for lines with a distance of about 4000 units would be 

					
						(J+J_0)*4000

Transition allows to enter a transition for 1 to 6 quantum numbers (use + and - to alter the number of QNs). It is important to also check the according increase checkboxes (labelled with Incr), for example a-type transitions (in the typical SPFIT/SPCAT notation) can be followed by checking the first and third increase checkbox. They define which quantum numbers are increased in the series. If a transition is not found, the frequency will be set to 0.

If you struggle to set up the correct values for the Transition tab, use the pathfinder window and press Start on a transition, which will transfer the correct values to the Transition tab.

If you have already assigned transitions and are using the Transitions method, you can use the fitted frequency instead of the predicted frequency by checking Zero Assigned. This option is only viable in some rare cases and more gimmicky.

List allows for lists of frequencies, the list should contain frequencies that are separated by whitespace, comma or semicolon. Whitespace includes space, newline and tab. The provided lines will be sorted automatically in frequency. With Min Index you can set the first frequency that will be used. If the index of the plot is higher than the number of frequencies the plot will be set to a frequency of 0.

Apply updates the series definition and recenters the plots to the predicted frequency, as you are still able to move inside the plots to higher or lower frequencies.

If you follow a transition series but do not see predictions in the center of each plot, there is a nonzero offset. By pressing Apply the offset is set to zero and the predictions should now be in the center of your plots.

Navigating the Plots

Plots can be added or deleted with add plot and delete plot. With in and out the zoom can be altered and with left and right the current view can be shifted to lower or higher frequencies. By default, selecting a range of the plot with the mouse will zoom to this range. All these actions are also connected to keyboard shortcuts. Additionally, under Plot > Go to … the center frequency can be set manually and under Plot > Set Zoom a specific width can be set.

If you want to add or delete a lot of plots with huge datasets, it is usually faster to select the desired number of plots under Plot > # Plots instead of pressing add plot or del plot multiple times.

Assigning Lines and Assigned Lines

If you do not specify otherwise, you can fit a transition by pressing shift and selecting an according range with your mouse. Then the lineshape of the fit and the middle position will be indicated in the plot. The assigned line is added to the Assigned Lines table. The uncertainty will be set according to the default uncertainty value, which can be found in Configure Plots. Non negative integers are interpreted as numeric values while negative values have a special meaning. For 0 > x >= -1 the absolute value of obs-calc will be calculated and -1 > x >= -2 will prompt for user input.

The fit method can be changed under Fit > Change Function, which switches to the next mehtod or a specific method can be selected under Fit > Choose Fit Function.

Each assigned line can be seen in the Assigned Lines window. The values can be edited or the line may be deleted by selecting the whole row and pressing the delete symbol X in the top left of the window. Pressing the tight button in the top right corner resizes the columns to their content.

Further controls for assigned lines can be found under Lines. There the assigned lines can be saved in the .cat or .lin format, additionally one can choose if the lines should be appended to the chosen file or the file should be overwritten. Lastly, there is an option to delete all assigned lines.

Assigned lines are marked with a star symbol on top of the stick of the stick spectrum. The color of the stars can be changed under Plot > Edit Cat Files.

If Fit > Always Assign is checked, every time a range is selected, the results will be added to the Assigned Lines table. If not selected, the Fit > Manual Assign action can be used to add the last assignment to the Assigned Lines table.

Without pressing shift the program will zoom and with shift pressed the program will fit. This can be reversed in the config. Additionally, there is an option under Fit > Always Fit which allows to fit every time a range is selected and never zoom. This option is handy for even faster assignment.

Configure Plots

This tab controls how the plots are displayed. The scale of the experimental files and the cat files can be chosen independently. The options are Full Scale which uses the global minimum and maximum, Dynamic Scale which uses the minimum and maximum of the current view, meaning that all currently displayed data is considered, and Per Plot Scale scales each plots data individually.

Above the scale options is an input field for a numeric value, this can be used to zoom the current scale. This is useful, if you want to assign especially weak lines, that are hard to identify next to stronger lines. The zoom value does not have any function for the Per Plot Scale option, for the other two options it mutliplies the original range with -exp(zoomvalue). This allows for big changes in scale in the upper range of the complete range and fine changes in the lower range.

Furthermore the axes can be en- or disabled, the plots can be decoupled to allow for individual shifts and the plots can be annotated, displaying the current transition or frequency of each plot, depending on which series method is used.

If Show closest transition is checked, the closest transition to the cursor will be displayed, what might be especially useful for identifying transitions. To change the sensitivity see the config section.

The default error field sets the error each assigned line gets as a default value. Non negative integers are used as a numerical value, -1 specifies to calculate the absolute obs-calc value, -2 specifies to prompt for user input after every assignment.

The binning field sets the binning parameter, which specifies the number of datapoints subplot has to handle. If the number of datapoints in the subplots range is lower than the binning parameter, no downsampling occurs. Are there more datapoints in the range of the subplot than specified by the binning parameter, then the data in the subplots range is grouped into equidistant bins, with widths according to the binning parameter, and only each bins maximum value is displayed, resulting in an positive enveloping. If you set the binning parameter too low the quality of your plots may suffer, as information will get lost. If you set the value too high, matplotlib might get unresponsive.

In other words the binning parameter can be seen as your personal matplotlib trust parameter, as a higher value will transfer computing cost from the main program (by binning) to the matplotlib library (by drawing more points). A sensible lower minimum is the horizontal resolution of your screen, the maximum depends on how many datapoints you trust matplotlib to handle without getting unresponsive or running into problems. Additionally, the optimal value depends on the number of subplots you plan to display, as more subplots increase the stress on the underlying matplotlib library.

If still unsere, it is feasible to set the binning parameter to 2-10 times your displays horizontal resolution, as you will not benefit from signifcantly more pixels than your monitor can display. This however might not be the most performant choice.

The program remembers which subplot had the last interaction, e.g. a range was selected, a range was fitted or the subplot was clicked. This information is used, if the user navigates to a specific frequency with the Go to … action or when additional windows are opened. To choose another subplot, click it without dragging or assign a line.

More Features

Blend Window

When using the Transition method to define a series and selecting the Blend Dialog checkbox, everytime a peak is fitted, lines within the distance of the input field below the Blend Dialog are presented in a dialog and the user can choose which lines he wants to assign to the peak. The dialog will not be shown, if there is only a single predicted line in this range. This is useful for blends of series, but the width should be set to a sensible value.

Save Options

With Files > Save Options the current options can be saved into the LLWP.ini config file. If View > Save Files is checked, the program saves the currently loaded files and data from the Assigned Lines table and will load them at the next startup. You can also change the values in the LLWP.ini file by hand, refer to the Flags and View sections for more information.

Show Lineshape

Under Modules > Lineshape Window is the option to open a second plot window with lineshapes instead of a stick spectrum for the cat values. The last clicked plot will be opened. It automatically updates to the current view of the main window and allows for fitting like the main window. This is useful. to get an idea, of if a certain arrangement of sticks corresponds with a certain lineshape.

Blended Lines

Under Modules > Blended Lines is the option to open the current section of the plot and fit various lineshapes to it. Possible lineshapes are Gaussian, Lorentzian , Voigt and the first and second derivative of the previous. By clicking in the spectrum or by pressing Add Peak an additional peak can be added, the resulting sum is shown in blue and with Transp the transparency of the single lineshapes can be changed. A baseline polynom can be added and made visible with the checkbox. Resulting middle positions can be directly assigned with the Assign button. This is useful if a line is clearly blended, as it results in a more accurate center frequency (if performed in a sensible way).

Peakfinder Window

To find peaks in your experimental spectrum you can use the peakfinder window, which is found under Modules > Peakfinder. On top a plot is shown, to verify the found peaks. Beneath are the buttons to run the peakfinder, a button to export all found peaks, an uncertainty input and the Only unassigned Lines checkbox. If it is checked, all peaks that are within the uncertainty value from an allready assigned line are excluded from the output.

The arguments for the peakfinder are found beneath the buttons. Always check if the found peaks are sensible in the plot after running the peakfinder.

Matplotlib Toolbox

There is an option to show the matplotlib toolbox under View, if you want to fine tune the plots, save a picture of the figure or do any other matplotlib related tasks.

Manual Drawing

To reduce stress from your system, there is an option to disable automatic drawing of the plots under Plot > Automatic Draw and an option to draw manually Plot > Manual Draw. Then the canvas will be only redrawn on manual draw and not on each update to the plots. This might be especially handy for huge datasets or huge amounts of plots, as those can make the underlying matplotlib library slow.

If your experience suffers from poor performance, consider changing your binning parameter, if it is set to a value higher than the magnitude of your screen resolution, try reducing it.

Pathfinder [Experimental]

The pathfinder module is still in an early stadium, feedback is especially welcomed, see Feedback.

The pathfinder window, which can be found under Modules > Show Pathfinder helps to find a sensible starting point in your spectrum. It allows to find transitions from the .cat files within a user can specified range and an additional condition. The matches are ordered by intensity and shown in a table.

By clicking start in the according row, the transition is transferred to the main window. Please doublecheck, which QNs you want to increase, as the values for the increase checkboxes are guessed by the program and might be wrong. Also be aware, that the transition might be at the upper limit of your spectrum and therefore you might want to shift to lower QNs with Decr.

For the additional condition all properties of the .cat file can be used, as there are “freq”, “error”, “lgint”, “degfreed”, “elower”, “usd”, “tag”, “qnfmt”, ‘qnu1’, ‘qnu2’, ‘qnu3’, ‘qnu4’, ‘qnu5’, ‘qnu6’, ‘qnl1’, ‘qnl2’, ‘qnl3’, ‘qnl4’, ‘qnl5’, ‘qnl6’ and the operators “+”, “-”, “==”, “<”, “>”, “>=”, “<=” and all sensible combinations out of these.

Additionally there are three checkboxes, to limit on certain transitions for asymmetric tops, as other molecule classes are straight forward. If you select none, no filter will be applied, otherwise only selected transitions will be shown.

Series Fit[Experimental]

The series fit module is still in an early stadium, feedback as well as feature requests are especially welcomed, see Feedback.

The series fit module is found under Modules > Series Fit. If the series fit window is open, assigned lines go there instead of into the assigned lines window. This module allows to fit the line positions of a series by a function that can be chosen by the user.

On top is a table with the assigned lines, please note that it is important to enter the correct quantum numbers for the transitions. Underneath are fields to select the desired series. As the assigned lines will be compared against this selection, entries with no quantum numbers will not be used.

Underneath is a field for the function, that should be used, enter the quantum numbers with the same label that is used in the table header. For example to fit with a 2B*J function enter 2*qnu1*B. The fit button will start the procedure of fitting. Messages are displayed in the field at the bottom.

If fitting succeeded, the calculated frequencies will be automatically entered into the List tab of Define Series. By pressing apply in that tab, the newly calculated frequencies will be used and displayed.

If List is the active tab of Define Series, quantum numbers can not be assigned automatically, switch back to Transition or enter them manually.

Pipe

To allow for fitting the data and creating new predictions from inside the program, the pipe feature was introduced. Set up a pipe command under Modules > Set up Pipe, choose if the loaded files, being spectra, .cat and .lin files, should be reloaded after the command has finished, save the command and run it as a test.

The pipe command is the same command you would use from the command line to generate your predictions. In short, everything that goes in the shell, goes here. Newline characters are replaced by ’ && ’, allowing the user to use new lines for new commands.

Please keep in mind, that the working directory might differ from your usual routine. Maybe add the desired folders to your path environment variable or change your working directory by entering cd /path/to/my/folder before your pipe command .

Console Input

By pressing Shift+K a field opens, in which you can write python console code. This options is very powerful and is only useful, if you have some understanding of the underlying structure and a clear goal in mind.

Configuration

The configuration object can be inspected under View > Show Config. Not all their shown values are updated in realtime and certain values are only read on startup, meaning changing them life in the program has no effect. Press Refresh to update the config and press Save (or View > Save Options) to save the config. When saving the options a config file with the name LLWP.ini will be created, which can be edited either by hand or by changing the according value in the graphical user interface and saving afterwards. When changing values life in the config window, readonly lines are indicated by an (RO) suffix and a fixed label at the end of the line. When changing editable values the label at the end of the row indicates if the new value is understood or not. The following paragraphs describe the configuration parameters, their allowed values and meanings.

Only very few parameters of the config cannot be changed in the graphical user interface, e.g. the layout direction. Thus, beginners can mostly ignore the rest of this chapter. Also note, that certain self explaining parameters are not explained explicitly.

Available Versions

Two versions of the program are currently available, an .exe file and an .pyw file. The .pyw file is useful if you want to inspect or borrow code and has a faster startup. The .exe file is handy for all those, who despise the python package manager.

The .exe version is compiled to start without the console window popping up, the .pyw file does also start without a console window. If you want to see the python console, change the ending from .pyw to .py.

Tipps and Tricks

You can also add files with drag and drop, just release them over the program and choose the correct file class in the pop up dialog.

Instead of pressing the Incr and Decr buttons in theTransition tab of Define Series, you can also use your mousewheel, gone are the days of wound fingertips.

You assigned some beautiful transitions and accidentally closed the program or it crashed? In the first case you are lucky, everytime the program is shut down properly, a LLWP.cat file is saved with the transitions, that are currently in the Assigned Lines table. If it crashed gracefully you are lucky, as the program did also write a LLWP.cat file and an LLWP.err error report.

All fields with little arrows at the right hand side, indicating a numeric field, can also be altered with the mousewheel when selected and the Ctrl modifier will increase the step size, so before your mousewheel starts to get hot, try to use Ctrl

It is not recommended to set the number of subplots to an absurdly high number, as this action is not threaded, meaning it will block the other actions, and the underlying process of creating a figure with many subplots and the needed initial setup is quite slow. Everything in the range of low houndreds should be perfectly fine, but if you are keen to push the limits, please have first a dry run without any assigned lines, as on my machine 1000 subplots caused the program to become unresponsive for about 50 seconds, however no crashes occurred.

FAQ - Frequently Asked Questions

Can I doubleclick projects to open? Like in excel, I mean I am not a programmer, but it cant be that hard!!1!!

You can open projects by doubleclicking a project file on windows with the .exe version. To do so right click a .llwp project file and selct Open with and then Choose another app. Be sure to check Always use this app to open .llwp files.

For the .pyw version a little workaround is needed. Create a batch file in the same folder with the name LLWP.bat and the following content:

					
						cd %~dp0
						START .\LLWP.pyw %1

Then repeat the same steps as for the .exe version but choose the LLWP.bat file.

Please note, that using this technique will open a new instance on every doubleclick, if you want to open a project in an already running instance of LLWP you can drag and drop the project into this instance or use the Load Project action.
